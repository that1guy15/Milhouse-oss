# Code Of Conduct

Milhouse contributors should be direct, respectful, and oriented toward improving the system.

## Expected Behavior

- Assume operational failures are system failures until evidence shows otherwise.
- Critique work and process, not personal worth.
- Make feedback specific, actionable, and evidence-backed.
- Protect private data and user trust.
- Help maintainers reproduce issues.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Publishing private credentials, telemetry, or personal data.
- Deliberately bypassing security or privacy safeguards.
- Repeatedly derailing technical discussions.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; restrict participation; or block contributors when needed to keep the project healthy and safe.
