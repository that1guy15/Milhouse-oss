## Containment, portability, and failure injection

The canonical command is direct `./setup.sh`; explicit `sh setup.sh` or `bash setup.sh` invocation
is unsupported because it bypasses the sanitized shebang. Gate targets use
`./scripts/run_make.py TARGET`, which removes inherited Make and shell controls before Make parses
the repository file.

- A disposable fresh source copy ran `./setup.sh` twice under umask `000`, inherited `noexec`, and
  shell-startup controls. Both runs passed. The resulting environment contained 23,737 regular
  files, zero multiply linked files, three verified interpreter symlinks, and zero non-symlink
  entries with group/other permission bits.
- A network-disabled Debian 12 / CPython 3.11 container exercised Linux `/proc/self/fdinfo` mount
  identity. An ordinary tree passed, while a privileged same-device nested bind mount was rejected
  before traversal or mode mutation. Debian `dash` safely discarded an exported Bash function;
  explicit Bash preserved it and the first Python process rejected it before synchronization.
- Environment validation and permission restriction are descriptor-relative. The implementation
  holds the root and trusted interpreter descriptors, reopens every component without following
  links, compares complete child-name/identity snapshots, and mutates only a verified open inode
  with `fchmod`; it contains no pathname `chmod` fallback.
- Automated regressions swap the root, a nested directory, and a regular file immediately before
  descriptor mutation and prove every replacement/outside inode, mode, and byte remains unchanged.
  If a held regular file or directory is renamed outside during `fchmod`, the complete pinned
  ancestor/name binding fails immediate descriptor-relative verification and its prior mode is
  restored through the held descriptor; escaped originals and replacements remain byte- and
  mode-identical. A raced outward hardlink is likewise detected and rolled back. Additional cases
  cover missing/precreated roots,
  symlinks, hardlinks, special files, ownership, device and mount changes, interpreter targets,
  hostile umasks, shell options/functions, Make preloads, unsafe Make modes, and
  outside-working-directory cleanup.
- Artifact regressions replace a candidate during pinning and after inventory validation; private
  source/destination digests prevent mixed bytes, only the pinned originals reach install smoke,
  and changed public names fail final verification. A preplanted hash-manifest temporary symlink
  leaves its outside target byte- and mode-identical while publication fails closed.
- External link probes connect only to an address that passed public-address policy; redirects are
  resolved and checked independently while the original HTTP Host and TLS hostname remain intact.
- DCO regression evidence constructs an unsigned two-parent synthetic checkout merge: the exact PR
  base-to-head range passes while a range including the synthetic merge fails. Hosted pull requests
  bind the checker to `github.event.pull_request.base.sha` and `.head.sha` with full history. Pushes
  bind the complete `github.event.before`-to-`github.event.after` range; an unsigned earlier commit
  followed by a signed tip fails, while a fully signed multi-commit range passes.
- Workflow-policy regressions prove mandatory gates cannot acquire skip conditions and the aggregate
  cannot substitute hard-coded, swapped, duplicated, unused, missing, or extra result bindings.
