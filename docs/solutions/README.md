# Sanitized Engineering Solutions

This directory contains reusable, verified Milhouse engineering knowledge. It is not a session
archive, feedback store, telemetry store, incident dump, or substitute for the authoritative plan,
accepted ADRs, implementation-status evidence, provenance ledger, or runbooks.

A solution records its date, work package, gate, affected area, problem class, observable synthetic
failure, causal root, reusable pattern, regression evidence, applicability, exclusions, and related
contracts. It must be grounded in reviewed repository evidence and contain no raw prompts, responses,
transcripts, tool output, feedback bodies, logs, telemetry, provider payloads, credentials, private
identifiers, local paths, or private donor context.

Create a solution only for a verified technique or failure pattern that is reusable beyond one patch.
Route contract changes to a numbered amendment or ADR, gate evidence and defects to
`docs/implementation-status.md`, procedures to runbooks or troubleshooting, reuse facts to
`docs/provenance.md`, and vulnerabilities to private reporting.
