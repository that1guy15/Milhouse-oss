# Sanitized Solution Contract

Use a lowercase hyphenated filename under `docs/solutions/`. Include:

- title and date;
- work package, gate, and affected area;
- problem class and observable synthetic failure;
- causal root;
- reusable solution pattern;
- regression evidence with repository paths and concise command outcomes;
- applicability and when not to use the pattern;
- related plan clauses, ADRs, and superseded learning if any.

Do not include people, account or repository names beyond Milhouse OSS, machine paths, credentials,
private identifiers, raw quotes, production data, provider payloads, feedback bodies, prompts,
responses, transcripts, tool output, or session-derived summaries. A solution is durable engineering
knowledge, not a work log or session archive.
